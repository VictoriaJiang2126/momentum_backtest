# 动量突破策略回测项目设计文档 (v2 - 已确认参数)

## 一、项目目标

基于"捕捉Finviz突破型动量股"的交易策略，构建一个Python回测系统，验证策略在历史数据上的有效性，并输出可量化的绩效指标。

**核心问题：** 如果在过去7年中机械执行这套筛选+入场+止损规则，最终收益率、胜率、最大回撤是多少？是否优于直接持有SPY/IWM？

---

## 二、回测范围（已确认）

| 项目 | 设定 | 说明 |
|------|------|------|
| 时间范围 | 2018-01-01 至 2024-12-31 | 7年数据，覆盖2018Q4暴跌、2020疫情、2022熊市、2023-24牛市 |
| 股票池 | **Russell 3000 成分股** | ~3000只，覆盖全美约98%市值 |
| 初始资金 | $100,000 | |
| **单笔仓位** | **总资金的 20%** | **集中型组合** |
| **最大持仓数** | **5 只** | 满仓时100%投入 |
| 手续费 | 单边 0.05% | 模拟滑点+佣金 |
| 数据频率 | 日线（Daily OHLCV） | |
| 数据源 | yfinance | 接受幸存者偏差，结论中明确披露 |
| 回测引擎 | **vectorbt** | 见第五节技术选型说明 |

⚠️ **重要提醒：** 20% × 5 只是非常集中的组合，预期波动率会显著高于指数。单笔最大损失 = 20% × 7%(止损) = 总仓位 1.4%，数学可控但需做好心理准备。

## 三、策略规则

### 3.1 筛选条件（每个交易日收盘后扫描）

**必须全部满足：**

1. **流动性过滤**
   - 收盘价 > $5
   - 20日平均成交量 > 500,000 股

2. **趋势过滤（多头排列）**
   - 收盘价 > SMA20
   - SMA20 > SMA50
   - SMA50 > SMA200

3. **位置过滤**
   - 当前价格距离52周最高价 ≤ 10%

4. **动量过滤**
   - 过去5个交易日涨幅 ∈ [+10%, +20%]
   - （>20% 视为已经追高，<10% 视为动量不足）

5. **量能确认**
   - 当日成交量 / 20日均量 > 2.0（Relative Volume）

### 3.2 入场规则

满足筛选条件的股票，**第二个交易日以开盘价买入**（避免lookahead bias）。

**候选股票排序（关键，因为只有5个槽位）：**
- 按 Relative Volume 降序排列
- 优先买入量能最爆发的，直到5个槽位用完
- 已持仓的股票不会被重复买入

### 3.3 出场规则（任一触发即平仓）

| 优先级 | 条件 | 类型 |
|------|------|------|
| 1 | 收盘价 < 入场价 × 0.93 | 硬止损 -7% |
| 2 | 收盘价跌破10日均线 | 趋势止损 |
| 3 | 从持仓期间最高点回撤 ≥ 10% | 移动止盈 |
| 4 | 持仓超过30个交易日 | 时间止损 |

出场在**信号触发次日开盘价**执行。

### 3.4 仓位管理（已确认）

- 单只股票仓位 = 账户净值 × **20%**
- 同时最多持仓 **5** 只
- 满仓时几乎100%投入，几乎不留现金缓冲
- 已持仓股票不会被重复买入
- 如果当日有>5个候选但仓位已满，按 RVol 排序丢弃低优先级信号

## 四、数据源与股票池

### 4.1 数据源
- **主选：** `yfinance`
- 调用 `yf.download(tickers, start, end)` 批量下载
- 本地缓存为 parquet 文件，避免重复下载

### 4.2 Russell 3000 成分股获取
- 从 iShares Russell 3000 ETF (IWV) 公开持仓数据获取
- 备选：FTSE Russell 官方成分股列表
- ⚠️ 这里只能拿到**当前**成分股，不是历史成分股

### 4.3 ⚠️ Russell 3000 + yfinance 的幸存者偏差风险

**这是本回测最大的真实风险，必须承认：**

- Russell 3000 包含大量小盘股，2018-2024 期间约有 **15-25%** 的成分股被删除（因为退市、并购、跌破市值门槛）
- yfinance 不返回已退市股票数据
- 等于回测时**只看到了"活下来的赢家"**，回测结果会**系统性虚高**
- 真实策略收益率可能比回测结果低 **2-5%/年**

**缓解措施（项目内能做的）：**
1. 在最终报告中**明确标注此偏差**，告知真实表现会更差
2. 同时跑一个"S&P 500 only"的版本作为对照（S&P 500 偏差较小，约5-10%）
3. 检查每只股票的数据完整性：如果某股票2018年没有数据但2020年有，记录其上市时间，避免在不存在的日期产生信号

**根本解决（未来升级）：**
- 接入 CRSP / Polygon.io / Norgate Data 等付费数据源（包含退市股票）
- 这是专业回测的标配，但本期项目暂不做

## 五、技术栈与项目结构

### 5.1 为什么选 vectorbt 而不是 backtrader

| 维度 | vectorbt | backtrader |
|------|----------|-----------|
| 性能 | ✅ 向量化 + numba，3000股×7年≈几分钟 | ❌ 事件驱动逐日循环，可能跑几小时 |
| 横截面策略 | ✅ 原生支持（每天扫全市场） | ❌ 需要自己写复杂循环 |
| 学习曲线 | 中等（API较新但文档好） | 较平缓（更"传统"） |
| 出场逻辑灵活性 | 通过 SignalGenerator + custom callback 实现 | 直接写 next() 方法 |
| 社区/资源 | 中等 | 较丰富 |

**结论：** 这种"每日全市场扫描+排序选Top N"的策略，vectorbt 是正确选择。

## 六、回测核心逻辑（vectorbt 实现思路）

```python
# 伪代码 - 实际实现会更复杂

# Step 1: 一次性下载所有数据
prices = yf.download(russell_3000_tickers, start, end)  
# shape: (date, ticker, OHLCV)

# Step 2: 向量化计算所有指标（一次性，全市场）
sma_20  = prices.close.rolling(20).mean()
sma_50  = prices.close.rolling(50).mean()
sma_200 = prices.close.rolling(200).mean()
rvol    = prices.volume / prices.volume.rolling(20).mean()
high_52w = prices.close.rolling(252).max()
weekly_return = prices.close.pct_change(5)

# Step 3: 生成入场信号矩阵 (布尔矩阵 date × ticker)
entries = (
    (prices.close > 5) &
    (prices.volume.rolling(20).mean() > 500_000) &
    (prices.close > sma_20) &
    (sma_20 > sma_50) &
    (sma_50 > sma_200) &
    (prices.close >= high_52w * 0.9) &
    (weekly_return.between(0.10, 0.20)) &
    (rvol > 2.0)
)

# Step 4: 关键 —— 限制每日最多 5 个持仓
# 这是 vectorbt 中的难点，需要用自定义 order function:
#   每个交易日:
#     - 检查当前持仓数
#     - 如果 < 5，按 rvol 降序选 top (5 - 当前持仓数) 进入
#     - 否则跳过

# Step 5: 出场信号
exits = (
    price_below_stop_loss |
    price_below_sma10 |
    drawdown_from_peak_ge_10pct |
    holding_days_gt_30
)

# Step 6: 用 vectorbt 模拟（注意 shift 防 lookahead）
portfolio = vbt.Portfolio.from_signals(
    close=prices.open.shift(-1),   # 次日开盘成交
    entries=entries,
    exits=exits,
    size=0.20,
    size_type='percent',
    init_cash=100_000,
    fees=0.0005,
    freq='1D',
)

# Step 7: 输出指标
print(portfolio.stats())
portfolio.plot().show()
```

**实现难点：** vectorbt 的 `from_signals` 默认每只股票独立模拟，**不天然支持"全市场只持5只"的约束**。需要用 `from_orders` + 自定义 `order_func_nb`（numba 加速函数）来实现跨股票的资金共享和持仓数限制。这部分会是 Phase 3 的核心工作，可能需要参考 vectorbt 官方的 "Portfolio Optimization" 示例代码。

## 七、绩效指标

vectorbt 自带很多指标（`portfolio.stats()`），我会在此基础上补充：

**收益类**
- 总收益率、年化收益率（CAGR）
- 与 SPY、QQQ、IWM（Russell 2000，最接近的基准）对比

**风险类**
- 最大回撤 + 持续天数
- 年化波动率、夏普比率、索提诺比率、Calmar 比率

**交易类**
- 总交易次数、胜率、盈亏比、期望值
- 平均/中位数持仓天数
- 最大单笔盈利/亏损
- 连续亏损次数最大值
- **按年份拆分表现**（看是否依赖某个特定年份）

**专门为集中组合补充：**
- 资金利用率（平均持仓数 / 5）
- 单股最大贡献占比（避免靠一只股票拉收益的虚假繁荣）

**输出形式**
- 控制台打印汇总表
- HTML 报告（vectorbt 自带 + 自定义图表）
- CSV 导出全部交易记录

## 八、关键风险与注意事项 ⚠️

### 8.1 幸存者偏差（最大风险）
- 已在第 4.3 节详述
- 输出报告时会明确标注

### 8.2 前视偏差（Lookahead Bias）
- 严格规定：t 日的信号 → t+1 日开盘成交
- vectorbt 默认信号 t 日产生 → t 日收盘成交，需手动 shift(1)
- 计算指标时只用 t 日及之前数据
- 写专门的单元测试验证这一点

### 8.3 集中度风险
- 5 只持仓意味着单股权重高，回撤可能显著高于分散组合
- 报告中会单独列出"最差的 5 个交易日净值变化"

### 8.4 流动性假设
- 假设下单不影响价格
- Russell 3000 中很多小盘股日均成交额 < $1M，下单 $20K 可能就是 2% 的日成交量
- **缓解：** 检查每笔模拟交易是否超过当日成交量的 1%，超过的丢弃

### 8.5 财报跳空风险
- 持仓时遇到财报，可能跳空跌破止损
- 模拟出场价：`min(开盘价, 止损价)`，模拟最坏成交

### 8.6 参数过拟合
- 不要为了好看反复调参
- **样本外测试计划：** 用 2018-2022 跑出最优参数，2023-2024 作为 out-of-sample 验证
- 做参数敏感性分析：止损从 -5% 到 -10% 各跑一遍，看曲线是否平滑（平滑=稳健，剧烈变化=过拟合）

### 8.7 市场状态依赖
- 动量策略普遍在牛市好、熊市/震荡市差
- 报告中按年份拆分收益，看是否完全依赖 2020、2023 这种大牛市
- 重点关注 2022 年（熊市）的表现

## 十二、最后一个想跟你确认的小问题

在开始写 Phase 1 之前，还想问一下：

**问题：** 你希望最终交付物是哪种形式？

- (A) **可复现的脚本** —— `python main.py` 一键跑出报告，不交互
- (B) **Jupyter Notebook** —— 分步骤执行，可以中途调整参数看效果  
- (C) **两者都要** —— 核心代码模块化，再用 notebook 做分析展示

我的推荐是 **(C)**：模块化代码用于稳定回测，notebook 用于探索和参数实验。这样你既能跑标准回测，也能交互式分析"如果止损改成 -5% 会怎样"这种问题。
